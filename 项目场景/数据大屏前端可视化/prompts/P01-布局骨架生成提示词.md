## 🎯 G - 目标与意图

### 核心目标

生成一个**结构清晰、可扩展、响应式**的大屏布局骨架，为后续组件开发提供稳固基础。

### 具体目标

1. **结构化**: 使用CSS Grid实现精确的多列布局，区块划分清晰
2. **可扩展**: 每个区块预留具名插槽(slot)，支持组件热插拔
3. **主题化**: 所有颜色、间距使用CSS变量，支持主题切换
4. **响应式**: 支持多种分辨率自适应（1080p/2K/4K）

### 业务价值

- **为开发团队**: 提供统一的布局标准，避免各自为战导致的样式混乱
- **为后续迭代**: 新增/删除区块只需修改配置，无需重构布局
- **为性能优化**: 布局层与内容层分离，支持按需加载

### 成功标准

- ✅ 布局在目标分辨率下完美填充，无滚动条
- ✅ 所有区块边框、间距一致，符合设计规范
- ✅ CSS变量覆盖所有主题相关属性（颜色、边框、阴影）
- ✅ 每个区块有明确的slot名称和占位提示

## 📤 O - 输出要求

### 1. 输出结构

```
src/
├── layouts/
│   └── DashboardLayout.vue    # 主布局组件
├── styles/
│   ├── variables.scss         # CSS变量定义
│   └── dashboard.scss         # 大屏全局样式
└── types/
    └── layout.ts              # 布局相关类型定义
```

### 2. 质量要求

#### 代码规范
- 使用 `<script setup lang="ts">` 语法（Vue 3）
- 所有组件/变量使用语义化命名
- 关键代码添加注释说明

#### 样式规范
- 使用BEM命名规范（如 `.dashboard__sidebar`）
- 颜色必须使用CSS变量（如 `var(--color-primary)`）
- 避免魔法数字，间距使用变量（如 `var(--spacing-md)`）

#### 响应式规范
- 使用 `clamp()` 实现字体自适应
- 使用 `minmax()` 实现列宽弹性
- 提供多分辨率的媒体查询断点

### 3. 格式规范

- Vue SFC格式（template + script + style）
- SCSS嵌套层级不超过3层
- 每个文件添加文件头注释（说明用途、作者、日期）

### 4. 特别说明

#### 区块占位符设计

每个区块内需包含占位符，格式如下：

```vue
<div class="dashboard__block dashboard__block--{区块标识}">
  <div class="block__header">{区块标题}</div>
  <div class="block__content">
    <slot name="{区块标识}">
      <!-- 占位符：{组件描述} -->
      <div class="block__placeholder">📊 {组件名称}将在此处渲染</div>
    </slot>
  </div>
</div>
```

#### 全屏适配

- 使用 `100vw` / `100vh` 填充视口
- 禁用页面滚动（`overflow: hidden`）
- 处理浏览器默认边距（`margin: 0; padding: 0`）
