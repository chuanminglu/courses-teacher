# 数据大屏前端可视化 - 场景概述

> **📌 场景定位**: 企业级数据可视化大屏的AI辅助开发，适用于监控中心、运营大屏、IoT展示等场景。

---

## 🎯 适用场景

### ✅ 适合使用本场景模板

- 企业运营监控大屏（如：电力系统监控、网络运维中心）
- 数据中心资源监控（如：服务器状态、容量规划）
- 智慧城市/园区可视化（如：交通流量、环境监测）
- 工业IoT实时监控（如：生产线状态、设备健康度）
- 金融交易大屏（如：实时行情、风控指标）
- 物流调度中心（如：车辆追踪、仓储状态）

### ❌ 不适合使用本场景模板

- 后台管理系统（表格/表单为主）→ 使用「后台管理系统」场景
- 移动端H5应用 → 使用「移动端H5应用」场景
- 数据报表系统（导出/打印为主）→ 考虑报表工具

---

## 📊 核心特征

数据大屏与普通Web应用的关键差异：

| 维度 | 数据大屏 | 普通Web应用 |
|-----|---------|------------|
| **分辨率** | 固定分辨率（1080p/2K/4K） | 响应式多端 |
| **交互方式** | 以展示为主，轻交互 | 复杂表单、流程操作 |
| **数据更新** | 实时/准实时（WebSocket） | 用户触发刷新 |
| **视觉风格** | 深色科技风、数据密集 | 轻量化、留白多 |
| **组件类型** | 图表为主（ECharts/D3） | 表单/表格为主 |
| **性能关注** | 大量图表同时渲染 | 首屏加载、交互响应 |

## 🔧 技术栈推荐

### 默认技术栈（提示词模板基于此）

| 类别 | 技术选型 | 版本 | 说明 |
|-----|---------|------|------|
| 框架 | Vue 3 | 3.4+ | Composition API + `<script setup>` |
| 语言 | TypeScript | 5.0+ | 严格类型检查 |
| 图表 | ECharts | 5.5+ | 功能全面，社区活跃 |
| 样式 | SCSS | - | CSS变量 + Design Token |
| 状态 | Pinia | 2.0+ | Composition API风格 |
| 构建 | Vite | 5.0+ | 快速热更新 |
| 实时 | WebSocket | - | 原生或Socket.io |

### 替代技术栈

根据团队技术栈，可调整提示词中的技术要求：

| 场景 | 替代方案 |
|-----|---------|
| React技术栈 | Vue 3 → React 18 + Hooks |
| 阿里生态 | ECharts → AntV G2/G2Plot |
| 3D效果 | ECharts → Three.js + ECharts GL |
| 低代码 | 组件开发 → DataV/大屏设计器 |

## 🚀 快速开始

### 步骤1: 准备设计稿/截图

收集以下信息：
- [ ] 大屏设计稿或截图
- [ ] 目标分辨率（1920×1080 / 2560×1440 / 3840×2160）
- [ ] 色彩规范（主色、辅助色、背景色）
- [ ] 数据接口文档（可选，后期对接）

### 步骤2: 阅读方法论指南

详细阅读 [数据大屏AI代码生成指南-通用版.md](数据大屏AI代码生成指南-通用版.md)，理解：
- 分层渐进式生成方法
- 4个Phase的工作流程
- 组件分类与拆解策略

### 步骤3: 按顺序使用提示词

推荐生成顺序：

```
P02(设计系统) → P01(布局骨架) → P09(数据卡片) → 
P03(仪表盘) → P04(折线图) → P05(柱状图) → P06(环形图) → 
P07(地图) → P08(关系图) → P10(告警列表) → 
P11(组件组装) → P12(数据对接)
```

### 步骤4: 填充输入材料

每个提示词都有**输入材料模板**，根据实际项目填写：

```markdown
### 材料1：布局结构描述

【整体布局】
- 分辨率：{填写目标分辨率}
- 布局类型：{几列布局}
- 列宽比例：{每列占比}

【区块划分】
- 左侧栏（{xx}%）：{功能描述}
  - {组件1名称}
  - {组件2名称}
...
```

### 步骤5: 迭代验证

每生成一个组件：
1. 立即运行代码验证
2. 检查样式是否符合设计
3. 发现问题及时迭代优化
