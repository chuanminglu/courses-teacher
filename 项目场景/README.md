# 项目场景模板库

> **📌 说明**: 本目录收集真实项目中抽取的典型开发场景，每个场景包含完整的AI辅助开发提示词模板和方法论指南。

---

## 📚 目录说明

本目录按**开发场景**组织，每个场景是一个独立目录，包含：

```
场景名称/
├── README.md              # 场景概述与快速入门
├── {场景}指南-通用版.md    # 完整方法论与工作流程
└── prompts/               # 标准化提示词模板库
    ├── P01-xxx提示词.md
    ├── P02-xxx提示词.md
    └── ...
```

## 🎯 使用方式

### 方式一：按场景选用

1. 根据项目类型，进入对应场景目录
2. 阅读场景README了解适用范围
3. 按工作流顺序使用提示词模板

### 方式二：组合使用

不同场景的提示词可以组合使用，例如：

- **数据大屏** 的图表组件 + **后台管理** 的表格组件 = 可视化报表系统
- **RESTful API** + **数据大屏** = 全栈监控平台

### 方式三：定制扩展

1. 复制通用版提示词到项目特定目录（如 `prompts/前端可视化/`）
2. 根据实际需求填充输入材料
3. 保留定制版作为项目资产

## 📋 提示词规范

所有提示词遵循 **RTGO框架**：

| 要素 | 说明 | 必填 |
|-----|------|-----|
| **R** - Role | 角色定义：专业背景、核心能力 | ✅ |
| **T** - Task | 任务描述：输入材料、任务上下文 | ✅ |
| **G** - Goal | 目标意图：核心目标、业务价值、成功标准 | ✅ |
| **O** - Objective | 输出要求：结构、质量、格式、特别说明 | ✅ |

详细规范参见：[copilot-instructions.md](../.github/copilot-instructions.md)

## 🔧 贡献指南

### 添加新场景

1. 在本目录下创建场景目录（中文命名）
2. 创建 `README.md` 说明场景概述
3. 创建 `{场景名称}指南-通用版.md` 包含方法论
4. 在 `prompts/` 下创建提示词文件（P01-Pxx编号）
5. 更新本文件的场景分类表格

### 命名规范

- 场景目录：中文名称，如 `数据大屏前端可视化`
- 指南文件：`{场景}指南-通用版.md`
- 提示词文件：`P{序号}-{功能}提示词.md`
