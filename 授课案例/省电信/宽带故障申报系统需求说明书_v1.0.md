# 宽带故障申报系统需求说明书 v1.0

## 1. 需求概述

**需求编号**：REQ-20251126-001  
**需求来源**：客服中心/网络运维部  
**提出时间**：2025年11月26日  
**需求类型**：性能优化（提升现有功能效率）  
**原始描述**："宽带故障申报系统要优化一下，客户投诉处理慢"

**核心诉求**：优化宽带故障申报系统的处理效率，重点解决客服人员录入工单耗时长、工单派发等待时间长、维修工响应速度慢三大瓶颈，将平均故障处理周期从当前的24-48小时缩短至≤24小时，客户满意度提升至≥85分，降低因"处理慢"引发的客户投诉。

---

## 2. 业务背景

### 2.1 系统简介

**系统定位**：宽带故障申报系统是广东电信客户服务体系的核心系统之一，负责接收、处理、跟踪宽带网络故障申报全生命周期管理。

**核心功能**：
- 故障申报接入（电话、微信、官网等多渠道）
- 工单创建与派发（客服录入→调度中心→维修工）
- 现场维修与进度跟踪
- 客户反馈与满意度评价
- 故障数据分析与报表

**服务对象**：
- **主要用户**：广东电信宽带客户（家庭宽带、企业专线）
- **操作用户**：客服人员、调度管理员、维修工程师
- **管理用户**：客服中心主管、运维部门经理

### 2.2 当前痛点

基于问卷第2题，识别核心痛点：

- **痛点1：客服人员录入工单耗时长**  
  客服需要手动询问客户详细信息（账号、地址、故障现象、联系方式等），平均每个工单录入耗时5-8分钟，高峰时段客户等待时间长，引发不满。
  
- **痛点2：工单派发到维修工的等待时间长**  
  调度中心采用人工派单模式，需要逐个查看维修工位置、工作量、技能匹配度，平均派单耗时15-30分钟，紧急故障无法快速响应。
  
- **痛点3：维修工响应速度慢或上门时间长**  
  维修工接单后需要手动查找客户地址、拨打电话确认、规划路线，信息不对称导致平均上门时间延长1-2小时，客户焦虑等待。

### 2.3 优化目标

- **效率目标**：平均故障处理周期从当前24-48小时缩短至≤24小时（80%工单达标），紧急故障2小时内响应率≥95%
  
- **质量目标**：客户满意度从当前75-80分提升至≥85分，投诉率下降30-50%（从当前15%降至≤10%）
  
- **成本目标**：减少人工派单工作量60%，降低客服中心话务量（催单电话减少60%），提升维修工单日处理单量20%

## 4. 业务流程分析

### 4.1 现有流程（As-Is）

**传统故障申报处理流程**（平均耗时24-48小时）：

1. **客户电话申报**（5-10分钟）  
   客户拨打10000热线，语音导航后转人工，等待客服接听
   
2. **客服人工录入工单**（5-8分钟）  
   客服逐项询问：账号、姓名、地址、故障现象、联系方式、预约时间等，手动输入系统
   
3. **工单提交到调度中心**（15-30分钟等待）  
   工单进入待派发队列，调度员按接收顺序处理
   
4. **调度员人工派单**（15-30分钟）  
   调度员查看维修工位置、技能、工作量，手动选择合适维修工，电话/短信通知
   
5. **维修工手动操作**（30-60分钟）  
   维修工查看短信/系统→手动复制地址到地图→拨打客户电话→规划路线→出发
   
6. **上门维修**（2-4小时）  
   维修工到达现场，排查故障，修复，手动更新工单状态
   
7. **客服回访**（1-2小时）  
   客服致电客户确认故障是否解决，记录满意度评价

**现有流程痛点**：
- ⚠️ **人工操作环节多**：录入、派单、查询地址、拨号等多个环节依赖人工，效率低下
- ⚠️ **等待时间占比高**：工单排队、派单等待、维修工路上耗时占总周期的60-70%
- ⚠️ **信息不对称**：客户不知道工单进度，频繁催单；维修工不知道客户具体位置，浪费时间

### 4.2 优化后流程（To-Be）

**智能化故障申报处理流程**（平均耗时≤24小时，紧急故障≤2小时响应）：

1. **客户多渠道自助申报**（1-3分钟）  
   客户选择便捷方式申报：
   - 微信小程序扫码或公众号入口→填写预设表单（账号自动识别、地址自动定位）
   - 短信关键词申报（发送"故障+账号"到10000）
   - 官网在线工单提交（企业客户）
   - 传统10000热线（保留）

2. **系统自动创建工单**（5-10秒）  
   系统自动提取客户信息（账号关联姓名、地址、历史故障记录），生成结构化工单

3. **智能派单系统自动分配**（3-5秒）  
   AI算法根据以下规则自动派单：
   - 就近派单：GIS定位，匹配距离客户最近的维修工（缩短上门时间40%）
   - 技能匹配：根据故障类型（光猫、线路、路由器）匹配专业技能维修工（提升一次修复率30%）
   - 紧急优先：影响>100户的紧急故障自动标记为P0优先级，强制2小时内响应

4. **维修工移动端一键操作**（10-20秒）  
   维修工收到推送提醒（微信/短信/App）→点击工单→系统自动：
   - 一键导航到客户地址（调用高德/百度地图）
   - 一键拨号联系客户（免手动输入）
   - 显示故障详情和历史记录（辅助诊断）

5. **上门维修与自动更新**（1-3小时）  
   维修工到达现场→修复故障→App内点击"完工"→系统自动更新工单状态为"已完成"

6. **客户实时查询进度**（全流程）  
   客户在微信小程序查看工单状态：
   - 已受理：工单创建成功，预计XX时间内派单
   - 已派单：维修工XXX已接单，预计XX时间到达
   - 维修中：维修工正在处理
   - 已完成：故障已修复，请评价

7. **自动满意度评价**（1分钟）  
   工单完成后，系统自动推送满意度评价（微信/短信），客户点击评分

**优化效果预估**：

| 指标 | 优化前 | 优化后 | 提升幅度 |
|---|---|---|---|
| **平均处理周期** | 24-48小时 | ≤24小时 | **50%↓** |
| **工单录入时间** | 5-8分钟（人工） | 1-3分钟（自助） | **80%↓** |
| **派单时间** | 15-30分钟（人工） | 3-5秒（自动） | **99%↓** |
| **维修工接单响应** | 30-60分钟（手动操作） | 10-20秒（一键操作） | **95%↓** |
| **客户催单电话** | 每天约500通 | 每天约200通 | **60%↓** |
| **客户满意度** | 75-80分 | ≥85分 | **6-12%↑** |
| **紧急故障响应率** | 70%（2小时内） | ≥95%（2小时内） | **25%↑** |

### 5.1 功能详细说明

#### F-01：微信小程序自助申报

**功能描述**：  
客户通过微信扫描二维码或关注广东电信公众号，进入宽带故障申报小程序，填写预设表单提交工单。

**使用场景**：  
- 客户宽带无法上网，通过手机4G网络访问微信小程序申报
- 客户在微信公众号菜单栏点击"故障申报"→自动跳转小程序

**输入输出**：  
- **输入**：
  - 客户手机号（自动识别微信绑定号码）
  - 宽带账号（可选，系统根据手机号自动匹配）
  - 故障类型（下拉选择：无法上网、网速慢、频繁断网、其他）
  - 故障现象描述（可选文本输入）
  - 预约时间（可选）
  
- **输出**：
  - 工单编号（如：WD20251126001）
  - 预计处理时间（如：24小时内）
  - 进度查询入口（点击可查看实时状态）

**业务价值**：  
- 覆盖95%以上宽带客户（微信普及率高）
- 减少客服中心话务量30-40%（客户自助申报）
- 工单录入时间缩短80%（1-3分钟 vs 5-8分钟）
- 开发成本低（微信小程序开发周期约1-2个月）

#### F-02：短信关键词申报

**功能描述**：  
客户发送短信关键词到10000，系统自动识别并创建工单。

**使用场景**：  
- 老年客户不熟悉微信小程序，习惯使用短信
- 客户宽带故障导致无法上网，手机流量不足，无法使用小程序

**输入输出**：  
- **输入**：
  - 短信格式：`故障 [宽带账号]`（如：`故障 020-12345678`）
  - 或：`故障`（系统根据手机号自动匹配宽带账号）
  
- **输出**：
  - 自动回复短信：
    ```
    【广东电信】您的故障工单已创建，工单号：WD20251126001，
    预计24小时内处理，维修工将提前联系您。查询进度请回复：进度
    ```

**业务价值**：  
- 覆盖老年客户群体（约15-20%宽带用户）
- 操作极简，降低使用门槛
- 开发成本极低（短信网关接口开发约1周）

#### F-03：官网在线工单提交

**功能描述**：  
企业客户登录广东电信官网，在"企业服务"→"故障申报"模块提交工单。

**使用场景**：  
- 企业专线客户，需要提供详细故障描述和附件（如网络拓扑图）
- 企业IT管理员批量提交多个站点的故障工单

**输入输出**：  
- **输入**：
  - 企业账号登录（自动获取企业信息、专线列表）
  - 选择故障专线（下拉选择）
  - 故障类型（下拉选择）
  - 故障现象描述（文本框）
  - 附件上传（可选，支持图片、文档）
  
- **输出**：
  - 工单编号
  - 预计响应时间（企业客户SLA更高，如4小时内响应）
  - 工单跟踪链接

**业务价值**：  
- 覆盖企业客户（约10-15%宽带业务收入来源）
- 支持详细故障信息提交，提升一次修复成功率
- 符合企业客户管理习惯（PC端操作）

#### F-04：智能派单-根据维修工位置就近派单（GIS定位）

**功能描述**：  
系统实时获取所有维修工的GPS位置，计算与客户地址的距离，自动派发给距离最近的维修工。

**使用场景**：  
- 工单创建后，系统在3-5秒内自动完成派单
- 紧急故障优先派给最近的维修工，确保2小时内响应

**输入输出**：  
- **输入**：
  - 客户地址（经纬度坐标，通过地址解析API获取）
  - 维修工实时位置（维修工App后台上报GPS坐标）
  - 维修工状态（空闲/忙碌/离线）
  
- **输出**：
  - 自动选择距离最近的空闲维修工
  - 推送工单到维修工移动端
  - 预估上门时间（根据距离和实时路况计算）

**业务价值**：  
- 上门时间缩短40%（就近派单减少路程）
- 紧急故障响应率从70%提升至≥95%
- 减少调度员人工派单工作量90%

**技术实现**（业务需求层，不涉及技术细节）：  
- 维修工App定期上报GPS位置（每5分钟）
- 系统调用地图API计算距离和预估时间
- 优先匹配距离<5公里的维修工

#### F-05：智能派单-根据维修工技能匹配派单

**功能描述**：  
系统根据故障类型，自动匹配具备相应技能的维修工。

**使用场景**：  
- 光猫故障→派发给擅长光猫配置的维修工
- 线路故障→派发给擅长线路排查的维修工
- 路由器故障→派发给擅长网络设备调试的维修工

**输入输出**：  
- **输入**：
  - 故障类型（客户申报时选择或客服标注）
  - 维修工技能标签（系统预设：光猫、线路、路由器、综合）
  
- **输出**：
  - 自动选择技能匹配的维修工
  - 如无匹配的维修工，自动降级为"综合"技能维修工

**业务价值**：  
- 一次修复成功率提升30%（技能匹配减少二次派单）
- 减少客户等待时间（避免维修工到场后无法处理，需再次派单）
- 提升维修工工作效率（减少无效上门）

#### F-06：智能派单-紧急故障自动优先派发

**功能描述**：  
系统自动识别紧急故障（影响>100户），标记为P0优先级，强制2小时内派单并响应。

**使用场景**：  
- 小区机房光缆故障，影响500户客户
- 企业专线中断，影响企业运营
- 重要客户（VIP客户、政府单位）故障

**输入输出**：  
- **输入**：
  - 故障影响范围（系统根据故障地址自动关联受影响客户数）
  - 客户类型（VIP客户、企业客户、普通客户）
  
- **输出**：
  - 工单自动标记为"紧急"（红色高亮）
  - 强制派单规则：
    - 优先派给最近的维修工（距离<10公里）
    - 如维修工忙碌，自动发送紧急通知要求中止当前工单
    - 2小时内未响应，自动升级至主管处理

**业务价值**：  
- 保障SLA达成（紧急故障2小时内响应率≥95%）
- 降低重大故障投诉率（快速响应减少客户不满）
- 符合电信行业监管要求（工信部要求紧急故障快速响应）

---

#### F-07：维修工移动端-工单推送提醒

**功能描述**：

**数据加密**：  
- 客户个人信息（姓名、地址、电话）传输采用HTTPS加密
- 客户密码（如官网登录）采用SHA-256加盐哈希存储
- 维修工GPS位置数据加密存储，仅调度系统可访问

**权限控制**：  
- **客户**：仅可查询本人工单，不可查询他人工单
- **客服人员**：可查询和修改所有工单，操作记录留痕
- **维修工**：仅可查看已派发给自己的工单，不可查看未派发工单
- **调度管理员**：可查看所有工单和维修工状态，可手动调整派单
- **系统管理员**：可查看所有数据，操作需双因素认证

**操作审计**：  
- 记录所有工单状态变更（创建、派单、完成、取消）及操作人
- 记录客服人员修改工单的操作日志（修改内容、修改时间、修改人）
- 记录调度管理员手动派单的操作日志（派单理由、派单时间）
- 审计日志保留≥3年（符合电信行业监管要求）

**隐私保护**：  
- 维修工仅可查看必要的客户信息（姓名、地址、电话），不可查看账号密码
- 客户地址脱敏显示（如"广州市天河区XX路XX号"，不显示门牌号，需拨号确认）
- 满意度评价匿名化处理（维修工不可查看评价人身份）

  - 客户满意度提升→客户流失率降低→收入增加（难以量化）
  - 投诉率降低→监管处罚风险降低（难以量化）

### 8.3 风险提示

**风险1：业务部门接受度**  
- **风险描述**：客服人员习惯人工录入，抵触使用新系统；老年客户不会使用微信小程序
- **影响级别**：中（可能导致系统使用率低于预期）
- **缓解措施**：
  - 分阶段推广：第一阶段仅在1-2个客服中心试点，收集反馈后优化
  - 用户培训：组织客服人员、维修工、老年客户培训（视频教程+现场演示）
  - 激励机制：设置KPI考核，鼓励使用微信小程序申报（如每单奖励积分）
  - 保留传统方式：短期内保留10000热线人工申报，逐步过渡

**风险2：数据迁移与系统集成**  
- **风险描述**：现有系统（ITSM、CRM）数据格式不统一，集成难度大；历史工单数据迁移可能丢失
- **影响级别**：高（可能导致项目延期或数据丢失）
- **缓解措施**：
  - 提前进行数据梳理（识别数据格式、字段映射关系）
  - 制定数据迁移方案（分批迁移+数据校验）
  - 开发数据同步接口（新旧系统双向同步，确保数据一致性）
  - 设置回滚机制（如数据迁移失败，可回滚到原系统）

**风险3：外部依赖（地图API、短信网关）**  
- **风险描述**：高德/百度地图API调用限额不足，导致导航功能不可用；短信网关故障，导致推送失败
- **影响级别**：中（可能导致部分功能不可用）
- **缓解措施**：
  - 签订SLA协议（与地图服务商、短信服务商签订服务等级协议）
  - 多服务商备份（同时接入高德+百度地图，任一故障时自动切换）
  - 监控告警（实时监控API调用成功率，异常时自动告警）
  - 降级方案（如地图API不可用，维修工手动复制地址到地图App）

**风险4：智能派单算法准确率**  
- **风险描述**：AI算法派单不准确（如派错技能维修工、距离计算错误），导致客户投诉
- **影响级别**：中（可能导致客户满意度下降）
- **缓解措施**：
  - 算法测试与优化（使用历史工单数据进行回测，准确率≥90%再上线）
  - 人工兜底（调度管理员可手动调整AI派单结果）
  - 学习优化（根据维修工反馈和客户评价，持续优化算法）
  - 设置阈值（如距离>15公里，自动提示调度员人工确认）

**风险5：用户隐私泄露**  
- **风险描述**：客户个人信息、维修工GPS位置被泄露，引发投诉或法律风险
- **影响级别**：高（可能导致监管处罚或客户流失）
- **缓解措施**：
  - 数据加密（传输采用HTTPS，存储采用AES-256加密）
  - 权限控制（严格按角色分配权限，操作留痕审计）
  - 定期安全审计（每季度进行渗透测试和代码审计）
  - 合规认证（通过等保三级认证、ISO27001信息安全认证）
