# 📁 输出文件夹说明

此文件夹用于存放演练过程中生成的所有文档和原型文件。

---

## 📂 文件组织建议

请按以下结构组织你的输出文件：

```
outputs/
├── 01-需求澄清.md                    （步骤1生成的需求澄清文档）
├── 02-{系统名称}-SRS.md              （步骤2生成的需求规格说明书）
├── 03-{Story-ID}-用户故事.md         （步骤3生成的用户故事）
├── 04-{Story-ID}-页面元素分析.md     （步骤4生成的页面元素分析）
├── 05-{Story-ID}-交互逻辑设计.md     （步骤5生成的交互逻辑设计）
└── 06-{Story-ID}-原型/               （步骤6生成的HTML原型）
    ├── index.html
    ├── page2.html
    └── ...
```

---

## 📝 文件命名规范

### 1. 需求文档

- **格式**: `02-{系统名称}-SRS.md`
- **示例**: `02-访客管理系统-SRS.md`、`02-订单管理系统-SRS.md`

### 2. 用户故事

- **格式**: `03-{Story-ID}-用户故事.md`
- **示例**: `03-US-001-用户故事.md`、`03-US-002-用户故事.md`

### 3. 页面元素分析

- **格式**: `04-{Story-ID}-页面元素分析.md`
- **示例**: `04-US-001-页面元素分析.md`

### 4. 交互逻辑设计

- **格式**: `05-{Story-ID}-交互逻辑设计.md`
- **示例**: `05-US-001-交互逻辑设计.md`

### 5. HTML原型

- **格式**: `06-{Story-ID}-原型/`文件夹
- **示例**: `06-US-001-原型/`、`06-US-002-原型/`

## 🔍 人工审核要点

**重要**：AI生成的文档需要人工审核和优化！

### 步骤1：需求澄清
- ❓ 问题是否真正澄清了需求盲点？
- ❓ 是否遗漏了关键的非功能需求？
- ❓ 问题是否可以实际向客户提问？

### 步骤2：需求文档（SRS）
- ❓ 功能需求是否可测试？
- ❓ 非功能需求是否可量化？
- ❓ 是否有明显的逻辑漏洞？

### 步骤3：用户故事
- ❓ Story拆分粒度是否合理（2-5天完成）？
- ❓ 验收标准是否可验证？
- ❓ 优先级和依赖关系是否合理？

### 步骤4：页面元素分析
- ❓ 页面结构是否符合用户习惯？
- ❓ 表单字段是否遗漏？
- ❓ 字段验证规则是否完整？

### 步骤5：交互逻辑设计
- ❓ 异常流程是否考虑充分？
- ❓ 错误提示是否友好？
- ❓ 页面跳转逻辑是否合理？

### 步骤6：HTML原型
- ❓ 原型是否能正常运行？
- ❓ 交互功能是否完整实现？
- ❓ 样式是否在移动端正常显示？
