#### 材料1：业务背景

数据标注园区需要构建一个现代化的访客管理系统，实现访客全流程线上化管理，包括：

- 来访申请线上提交
- 审批流程自动化
- 现场确认与签到
- 访客权限自动同步（门禁系统、车辆道闸联动）
- 企业部门与访客人员管理
- 消息推送（公众号、短信）
- 访客数据报表与分析

#### 材料2：核心需求

**功能需求**：

1. 访客预约申请：支持访客通过微信公众号提交来访申请
2. 多级审批流程：被访人审批 → 部门主管审批 → 安保审批
3. 门禁权限同步：审批通过后自动下发门禁权限到海康威视门禁系统
4. 车辆道闸联动：支持访客车牌号自动识别和道闸开启
5. 现场签到：支持二维码扫码签到、人脸识别签到
6. 访客轨迹记录：记录访客进出时间、访问区域
7. 数据统计分析：访客量统计、部门访客排行、异常访客预警

**非功能需求**：

1. 性能：支持1000+访客同时预约和签到，高峰期响应时间 < 3秒
2. 可用性：系统可用率 ≥ 99.5%
3. 安全性：访客身份证号、手机号加密存储，符合等保三级要求
4. 集成性：必须与现有门禁系统（海康威视）、车辆道闸系统集成
5. 移动优先：支持微信公众号、企业微信、移动端操作

#### 材料3：技术约束

- 后端技术栈：Spring Boot + MySQL + Redis + RabbitMQ
- 前端技术栈：React（管理后台）、微信公众号开发（访客端）
- 门禁系统：海康威视门禁系统（HTTP API集成）
- 部署环境：阿里云（ECS + RDS + SLB）
- 开发周期：3个月MVP版本上线
